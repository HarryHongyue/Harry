# 安全基线

此基线涵盖 Harry 主站点及其链接的后端能力项目，特别是 PDF Reader 和 Aircargo EDI。

## 服务器安全

- 仅开放端口 22、80 和 443。
- 仅使用 SSH 密钥；禁用密码登录。
- 禁用 root SSH 登录。
- 使用普通用户而非 root 进行部署。
- 使用防火墙，如 `ufw`。
- 定期应用操作系统更新。

## 反向代理安全

- 通过 HTTPS 路由所有公共访问。
- 不要将后端服务端口直接暴露给公共互联网。
- 使用 Caddy 或 Nginx 作为唯一的公共入口点。
- 在代理层设置上传大小限制。
- 为 API 路由添加基本速率限制。
- 将后端容器保留在内部 Docker 网络上。

## 后端 API 安全

- PDF Reader 和 Aircargo EDI 上传端点必须限制文件大小。
- 验证文件扩展名和 MIME 类型。
- 不要信任用户上传的文件名。
- 将临时文件存储在 UUID/会话隔离目录下。
- 在每次请求后清理临时文件。
- 在生产环境中，CORS 必须是白名单。不要使用 `allow_origins=["*"]`。
- 不要在公共错误响应中暴露服务器路径或内部堆栈跟踪。
- 不要记录敏感文件内容。
- 在可行的情况下以非 root 身份运行容器。

## 机密信息

- 不要提交 `.env`。
- 不要提交 GitHub 令牌、API 密钥、数据库密码、私钥或生产机密。
- 为所需的环境变量提供 `.env.example`。
- 轮换任何曾经提交过的机密。

## 下载安全

- 将安装程序和扩展包放在 GitHub Releases 或对象存储中。
- 下载页面应显示版本、平台、大小和 sha256。
- 不要在 Harry 仓库中保留 `.exe`、`.zip`、`.dmg` 或浏览器包二进制文件作为长期资产。
- 将发布说明附加到发布资产上。

## 项目特定说明

### PDF Reader

- 为 OCR 工作负载设置代理和 API 上传限制。
- 在写入磁盘之前验证 PDF 输入。
- 使用随机会话目录并在 `finally` 块中清理它们。
- 将 CORS 限制为部署的前端域名。
- 考虑对 OCR 路由进行速率限制，因为 OCR 可能会占用大量 CPU。

### Aircargo EDI

- 将所有 AWB/HAWB PDF 视为不受信任。
- 防止生成的工件下载的路径遍历。
- 将生成的文件访问限制为运行/会话拥有的工件。
- 将本地优先的桌面工作流与公共 API 部署分开。

### ODE Solver

- 发布安装程序校验和。
- 将安装程序保留在发布版本或下载存储中，而非 Harry Git 历史记录中。

### SurpriseMe

- 在每次发布之前审查浏览器扩展权限。
- 发布浏览器特定的包校验和。
- 从项目页面或独立站点保持隐私说明可见。
